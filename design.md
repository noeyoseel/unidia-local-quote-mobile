# 유니디아 견적 플로우 — 모바일 인터페이스 설계

## 제품 방향

**유니디아 견적 플로우**는 상담사가 고객 문의 캡처를 받아 차량·계약 정보를 확인하고, 금융상품과 영업 조건을 조정한 뒤, 고객 전달용 견적 요약과 발송 문구를 완성하는 업무용 모바일 도구다. 화면은 세로형 9:16과 한 손 조작을 기본으로 하며, iOS 기본 앱처럼 명확한 계층, 넉넉한 터치 영역, 하단 중심의 주요 액션을 사용한다.

실제 캐피탈사 엑셀 파일의 수식과 Microsoft Excel 데스크톱 프로그램을 직접 조작하는 기능은 모바일 런타임에서 실행할 수 없으므로, 이번 앱은 **캐피탈사별 계산 규칙을 모사한 로컬 견적 엔진**으로 완결된 모바일 흐름을 제공한다. 향후 현업 엑셀 템플릿과 동일한 결과가 필요할 경우에는 Windows PC에 별도 견적 브리지 서비스를 두고 모바일 앱과 연결할 수 있도록 데이터 구조를 분리한다.

## 화면 목록

| 화면 | 주요 콘텐츠 | 핵심 기능 |
|---|---|---|
| 홈 | 오늘의 상담 현황, 새 견적 시작 카드, 최근 상담 3건 | 신규 견적 시작, 최근 기록 재열람 |
| 문의 캡처 | 이미지 선택 영역, 선택한 캡처 미리보기, 추출 진행 상태 | 사진 보관함 또는 파일에서 캡처 선택, AI 추출 시작 |
| 정보 확인 | 브랜드, 모델, 트림, 차량가, 기간, 약정 주행거리 입력 필드와 추출 신뢰도 | AI 결과 수정, 상담중 상태로 임시 저장 |
| 조건 설정 | 렌트·운용리스·금융리스 선택, 캐피탈사 선택, 할인·수수료·잔가 조정 | 상담사의 영업 판단 입력, 실시간 요약 확인 |
| 견적 결과 | 월 납입금·초기비용·만기 예상액, 고객용 견적 카드, 발송 문구 | 문구 복사, 견적 카드 공유, 최종 확정 |
| 상담 이력 | 상담중·완료 필터, 고객별 차량과 금액 목록 | 기록 검색, 상세 재열람, 이어서 편집 |
| 설정 | 계산 기준 안내, 데이터 보관 안내, 앱 정보 | 계산 가정 확인, 로컬 기록 초기화 |

## 핵심 사용자 흐름

### 신규 견적 생성

상담사는 홈의 **새 견적 만들기**를 누른다. 문의 캡처 화면에서 고객 문의 이미지를 선택하면 앱이 AI 구조화 추출을 수행하고, 결과를 정보 확인 화면에 자동 입력한다. 상담사는 추출 내용을 검수한 뒤 조건 설정 화면으로 이동하여 금융상품, 캐피탈사, 할인액, 추가 수수료율, 잔가 모드를 입력한다. **견적 산출**을 누르면 결과 화면에서 월 납입금과 초기비용, 고객용 견적 카드, 발송 문구를 확인한다. 최종 확정 시 기록은 완료 상태로 로컬 보관된다.

### 상담중 견적 이어서 처리

홈 또는 상담 이력에서 상담중 기록을 선택한다. 이전에 입력한 차량 정보와 조건을 불러와 수정한 뒤 다시 견적을 산출한다. 확정하지 않은 기록은 상담중으로 유지되며, 확정한 기록은 완료 목록으로 이동한다.

### 고객 전달

결과 화면의 **문구 복사**를 누르면 인사말, 핵심 견적 요약, 필수 안내 문구가 클립보드에 복사된다. **견적 공유**를 누르면 모바일 공유 시트가 열리고 카카오톡, 텔레그램 등 설치된 앱을 선택할 수 있다.

## 화면별 레이아웃

홈 상단에는 작은 브랜드 워드마크와 날짜를 두고, 그 아래에 상담 현황을 두 개의 간결한 지표 카드로 표시한다. 가장 중요한 **새 견적 만들기**는 엄지손가락이 닿기 쉬운 화면 하단 절반에 큰 카드 버튼으로 배치한다. 최근 상담은 하단 탭 위에서 스크롤 가능한 리스트로 제공한다.

신규 견적 흐름은 각 화면 상단에 `1/4 문의 캡처`, `2/4 정보 확인`처럼 진행 단계를 표시한다. 주요 완료 버튼은 화면 하단의 고정 영역에 배치하되, 키보드가 열리면 입력 필드를 가리지 않도록 안전 영역과 키보드 회피 동작을 적용한다. 숫자 입력은 우측 정렬하고 원·개월·% 단위를 입력 필드 바깥에 명시한다.

견적 결과 화면의 첫 영역에는 고객이 가장 먼저 확인할 **월 납입금**을 크게 표시하고, 아래에 초기비용과 만기 정보를 2열 카드로 정리한다. 고객 전달용 견적 카드는 밝은 표면색, 얇은 구분선, 큰 금액 타이포그래피를 사용한다. 금융소비자 보호 관련 문구는 과도하게 숨기지 않되, 보조 계층의 작은 본문으로 표시한다.

## 색상과 시각 언어

| 용도 | 색상 | 적용 방식 |
|---|---|---|
| 브랜드 네이비 | `#10233D` | 상단 제목, 핵심 텍스트, 앱 아이콘 바탕 |
| 액션 블루 | `#2F6BFF` | 주요 버튼, 선택 상태, 진행 표시 |
| 민트 포인트 | `#24B38E` | 완료 상태, 절감액, 긍정 피드백 |
| 배경 | `#F4F6FA` | 전체 화면 배경 |
| 카드 표면 | `#FFFFFF` | 입력 카드, 견적 카드, 목록 행 |
| 기본 텍스트 | `#132033` | 제목과 본문 |
| 보조 텍스트 | `#657083` | 설명, 메타 정보, 안내 문구 |
| 경고 | `#D97706` | 필수 검토, 추출 신뢰도 낮음 |
| 오류 | `#D93F4C` | 입력 오류, 추출 실패 |

모서리는 14~20pt 범위로 제한하고, 그림자는 거의 사용하지 않으며 경계선과 표면색 차이로 계층을 만든다. 주요 버튼은 최소 52pt 높이, 보조 버튼은 최소 44pt 터치 영역을 확보한다. 아이콘은 단순한 선형 또는 채움형 시스템 아이콘만 사용하고 장식용 그래픽은 최소화한다.

## 데이터 모델

| 엔터티 | 핵심 필드 | 상태 |
|---|---|---|
| 상담 기록 | ID, 생성일시, 수정일시, 고객 식별 메모, 원본 이미지 URI | 상담중, 완료 |
| 차량 정보 | 브랜드, 모델, 트림, 차량가, 계약기간, 약정 주행거리, AI 신뢰도 | 확인 전, 확인 완료 |
| 견적 조건 | 상품 유형, 캐피탈사, 할인액, 추가 수수료율, 잔가 모드 | 편집중, 산출 가능 |
| 견적 결과 | 월 납입금, 초기비용, 잔존가치, 총 납입 예상액, 발송 문구 | 임시, 확정 |

앱 기록은 기기 로컬 저장소에 보관한다. 이미지 자체는 장기 복제하지 않고 기기에서 선택한 URI를 참조하며, AI 추출이 실패해도 수기 입력으로 전체 흐름을 완료할 수 있도록 한다.

## 계산 및 AI 처리 원칙

문의 캡처 분석에는 비용과 속도의 균형이 좋은 멀티모달 모델을 사용하고, 브랜드·모델·트림·차량가·계약기간·약정 주행거리·신뢰도를 엄격한 JSON 구조로 반환한다. 추출 결과는 자동 확정하지 않고 반드시 상담사가 검수한다.

견적 계산은 상품 유형, 캐피탈사 계수, 차량가에서 할인액을 차감한 취득원가, 기간, 수수료율, 잔가율을 이용한다. 화면에는 결과가 **예상 시뮬레이션**이며 실제 캐피탈사 승인·금리·세금·보험 조건에 따라 달라질 수 있음을 명시한다.
