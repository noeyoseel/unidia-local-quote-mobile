# Windows Excel 브리지 확장 경계

## 현재 앱의 책임

현재 모바일 앱은 문의 캡처 선택, AI 구조화 추출, 상담사 검수, 금융사·상품 선택, 할인·수수료·잔가 조정, 결정론적 견적 시뮬레이션, 고객 발송 문구 생성과 기기 로컬 아카이빙을 담당한다. 계산 결과에는 `engineVersion`과 입력 스냅샷을 함께 저장해 동일 조건을 재현할 수 있도록 한다.

## 실제 Excel 연동이 필요한 이유

Microsoft Excel 원본 파일의 수식, 매크로, 외부 연결과 인쇄 영역을 그대로 실행하려면 Windows PC의 Excel 데스크톱 프로세스가 필요하다. Expo 모바일 런타임이나 서버리스 Node 런타임에서 Excel COM 자동화를 직접 실행하지 않는다.

## 권장 구성

Windows 상담 PC에 별도 로컬 브리지 서비스를 설치한다. 브리지는 `127.0.0.1` 또는 사내망의 제한된 포트에서만 수신하며, 캐피탈사와 상품 ID에 따라 승인된 Excel 템플릿을 선택한다. 모바일 앱은 브리지와 직접 통신하기보다 사내 인증 프록시 또는 QR 페어링 토큰을 통해 요청을 전달하는 구성을 권장한다.

| 구간 | 책임 |
| --- | --- |
| 모바일 앱 | 입력 수집, 검수, 견적 요청 생성, 결과 표시 및 아카이빙 |
| 인증 프록시 | 사용자·기기 확인, 단기 토큰 발급, 감사 로그 기록 |
| Windows 브리지 | 템플릿 선택, 지정 셀 입력, Excel 재계산 대기, 출력 영역 내보내기 |
| Excel 템플릿 | 캐피탈사 공식 수식, 승인된 셀 매핑, 인쇄 레이아웃 |

## 요청 계약 예시

```json
{
  "quoteId": "local-uuid",
  "templateId": "capital-product-version",
  "vehicle": {
    "brand": "현대",
    "model": "그랜저",
    "trim": "캘리그래피",
    "price": 52000000
  },
  "terms": {
    "months": 48,
    "annualMileageKm": 20000,
    "discountAmount": 2500000,
    "feeRate": 2.5,
    "residualMode": "maximum"
  }
}
```

브리지 응답에는 `monthlyPayment`, `initialCost`, `residualValue`, `templateVersion`, `calculatedAt`과 PNG/PDF 결과 파일의 단기 접근 URL 또는 Base64가 포함되어야 한다.

## 안전 규칙

브리지는 임의 파일 경로, 임의 시트명 또는 임의 셀 주소를 요청에서 받지 않는다. 템플릿과 셀 매핑은 브리지 내부의 승인 목록으로만 관리한다. 요청별 작업 디렉터리를 사용하고, 완료 후 원본 고객 이미지와 임시 Excel 파일을 즉시 정리한다. 매크로는 서명된 승인 템플릿에서만 허용하고, 결과에는 템플릿 버전과 입력 해시를 남긴다.

## 장애 처리

Excel 실행 실패, 수식 오류, 재계산 시간 초과 또는 출력 영역 누락 시 앱은 기존 로컬 시뮬레이터 값을 공식 견적으로 오인해 확정하지 않는다. 브리지 결과에 `verified: true`가 있을 때만 “Excel 검증 완료” 상태로 전환하고, 실패 시 재시도와 수기 확인 안내를 제공한다.
